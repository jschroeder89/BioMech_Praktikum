20 /**
21  * @file chbsem.h
22  * @brief Binary semaphores structures and macros.
23  *
24  * @addtogroup binary_semaphores
25  * @details Binary semaphores related APIs and services.
26  * <h2>Operation mode</h2>
27  * Binary semaphores are implemented as a set of inline functions
28  * that use the existing counting semaphores primitives. The
29  * difference between counting and binary semaphores is that the
30  * counter of binary semaphores is not allowed to grow above the
31  * value 1. Repeated signal operation are ignored. A binary
32  * semaphore can thus have only two defined states:
33  * - <b>Taken</b>, when its counter has a value of zero or lower
34  * than zero. A negative number represent the number of threads
35  * queued on the binary semaphore.
36  * - <b>Not taken</b>, when its counter has a value of one.
37  * .
38  * Binary semaphores are different from mutexes because there is no
39  * concept of ownership, a binary semaphore can be taken by a
40  * thread and signaled by another thread or an interrupt handler,
41  * mutexes can only be taken and released by the same thread. Another
42  * difference is that binary semaphores, unlike mutexes, do not
43  * implement the priority inheritance protocol.<br>
44  * In order to use the binary semaphores APIs the
45  * @p CH_CFG_USE_SEMAPHORES option must be enabled in @p chconf.h.
46  * @{
47  */
